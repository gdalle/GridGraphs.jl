GridGraphs.AbstractGridGraph — Type
AbstractGridGraph{T<:Integer,R<:Real}

Abstract supertype for grid graphs with vertices of type T and weights of type R.

All subtypes must have a field weights::Matrix{R}, whose size gives the size of the grid and whose entries correspond to vertex weights. The weight of an edge (s,d) is then defined as the weight of the vertex d.

To implement a concrete subtype G <: AbstractGridGraph, the following methods need to be defined (see the Graphs.jl docs):

  • Graphs.ne(g::G)
  • Graphs.has_edge(g::G, s, d)
  • Graphs.outneighbors(g::G, s)
  • Graphs.inneighbors(g::G, d)

GridGraphs.AcyclicGridGraph — Type
AcyclicGridGraph{T<:Integer,R<:Real}

Concrete subtype of AbstractGridGraph, in which we can move from a cell (i,j) to its bottom, right and bottom right neighbors only. This means the graph is acyclic.

Fields

  • weights::Matrix{R}: grid of vertex weights
source
GridGraphs.GridGraph — Type
GridGraph{T<:Integer,R<:Real}

Concrete subtype of AbstractGridGraph, in which we can move from a cell (i,j) to any of its 8 nearest neighbors (lateral, vertical and diagonal).

Fields

  • weights::Matrix{R}: grid of vertex weights

GridGraphs.grid_dijkstra! — Method
grid_dijkstra!(queue, g, s; naive)

Apply Dijkstra's algorithm on an AbstractGridGraph g, and return a ShortestPathTree with source s.

The first argument is the priority queue used (and modified) during the algorithm. When the keyword argument naive is set to true, priority updates are disabled and vertices may instead be inserted several times into the queue.
